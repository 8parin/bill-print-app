
1. Multi-platform CSV (Shopee / Lazada / TikTok Shop)
Recommended approach: Platform presets + auto-detection

The current architecture already has a flexible column mapping system. Build on it:

Add platform presets — each platform (Shopee, Lazada, TikTok) gets its own default COLUMN_MAP with the correct Thai column names. User picks the platform on upload (or it auto-detects based on distinctive column names).
Auto-detection — each platform has unique columns. For example, Shopee has หมายเลขคำสั่งซื้อ, Lazada likely has different column names. Check the first row and match against known patterns.
Shared pipeline — after mapping, everything flows through the same Invoice data model. The parser normalizes platform-specific data into the common structure. Bill generation, printing, and sales report stay untouched.
Cancelled/return status — each platform uses different status values and column names. The preset would include platform-specific CANCELLED_STATUSES and CONFIRMED_RETURN_STATUSES.
Manual mapping fallback — the existing column mapping UI already handles unknown formats. If auto-detection fails, user maps manually.
This is clean because the only platform-specific code lives in the parser layer. Everything downstream is platform-agnostic.

2. Deployment: Local ZIP vs Remote Server
Recommended: Stay with local ZIP, but add Windows support

Local ZIP	Remote Server
Privacy	Data stays on machine	Sensitive order data over internet
Setup	One-time install	Needs hosting, domain, SSL
Cost	Free	Monthly server cost
Reliability	Works offline	Depends on internet
Multi-user	One user at a time	Multiple users
For your use case (bill printing from CSV exports), local ZIP is better because:

Order data is sensitive (customer names, addresses, phone numbers)
No ongoing hosting costs
Works without internet
Simpler to maintain
For Windows support, you'd need:

install.bat and run.bat (equivalents of the .sh scripts)
Fix the Thai font path (macOS: /System/Library/Fonts/.../Tahoma.ttf, Windows: C:\Windows\Fonts\tahoma.ttf) — add OS detection
The rest (Python, Flask, ReportLab, pandas) is already cross-platform
The ZIP would contain both .sh and .bat scripts. The user runs whichever matches their OS.
